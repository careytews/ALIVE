ParticipantPropertyFrequency (Int32)
This is an integer that specifies how often the daemon will send participant property events while in a channel. If this is not set the default will be “on state change”, which means that the events will be sent when the participant starts talking, stops talking, is muted, is unmuted. The valid values are: 0 – Never 5 – 10 times per second 10 – 5 times per second 50 – 1 time per second 100 – on participant state change (this is the default)
